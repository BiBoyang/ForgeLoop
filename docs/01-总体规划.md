# 总体规划（KWWK Swift 复刻）

## 目标
- 复刻与 KWWK 一致的核心能力，优先保证：
  - Harness Agent 回合控制与事件模型一致；
  - TUI 流式输出为“覆盖更新”而非简单追加；
  - 工具调用与上下文顺序一致（tool_call 前后关系正确）。

## 分层边界
- `ForgeLoopAI`：Provider、SSE、消息流抽象。
- `ForgeLoopAgent`：Agent 状态机、AgentLoop、工具执行、队列。
- `ForgeLoopCli`：输入、渲染、订阅事件、交互行为。

## 里程碑
- M1：Agent 主循环可跑通（FauxProvider + prompt + 流式）。
- M2：TUI 流式覆盖渲染稳定。
- M3：工具调用（至少 read/write/bash stub）+ 事件完整。
- M4：真实 provider 接入（OpenAI Responses 先行）。
- M5：收敛测试、交互打磨、文档补齐。
