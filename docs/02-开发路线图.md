# 开发路线图

## Phase 1：内核闭环
- `ForgeLoopAI`：`AssistantMessageStream`、`SSEParser`、`FauxProvider`。
- `ForgeLoopAgent`：`AgentState`、`Agent`、`AgentLoop`。
- 验证：`prompt -> streaming -> messageEnd` 完整链路。

## Phase 2：Harness 能力补齐
- `continue()`、`abort()`、队列（steering/follow-up）。
- 工具调用协议与事件（start/update/end）。
- 验证：并发保护、事件顺序、abort 行为。

## Phase 3：TUI 流式一致性
- `TranscriptRenderer`：`streamingRange` 覆盖替换。
- `TUI`：稳定重绘，避免重复行与闪烁。
- 验证：长文本流式、tool block 占位替换、resize。

## Phase 4：真实 Provider + 工具集
- OpenAI Responses（优先）。
- 最小工具集：`read`、`write`、`bash`。
- 验证：tool_call 顺序正确，回合可继续。

## Phase 5：收尾
- 测试补全、文档同步、版本标记。
