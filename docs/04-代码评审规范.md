# 代码评审规范（我执行）

## 评审维度
- 架构一致性：是否符合三层边界（AI/Agent/CLI）。
- 行为一致性：是否保持回合和事件语义。
- 并发安全：锁/actor/Task 使用是否避免竞态。
- 可维护性：命名、注释、复杂度是否可控。
- 可验证性：是否有可运行的最小验证步骤。

## 必查清单（Harness Agent）
- 是否保证单次 active run（防重入）。
- `messageStart -> messageUpdate -> messageEnd` 是否闭合。
- assistant(tool_call) 是否在 tool_result 之前进入上下文。
- abort 后状态是否回收（isStreaming、waiters）。

## 必查清单（TUI 流式）
- 是否覆盖更新（replace）而非重复 append。
- 流式结束后是否清理 streamingRange。
- tool 执行占位与最终结果是否正确替换。
- 输入行为是否区分 idle 与 streaming。

## 评审结论模板
- `Pass`
- `Changes Requested`（必须附最小修改建议）
